# CBIT128 relay control patterns

适用场景：
- 适配器继电器控制
- 开短路 / CON 测试前后的通道切换
- 多工位共享继电器矩阵控制

板卡能力要点：
- 128 路控制位
- 继电器驱动电压范围 5V~12V
- 负载能力示意：
  - +5V：100mA（继电器电源）
  - +12V：2A（继电器电源）
  - ±15V：500mA（模拟电源）

核心动作：
- `SetOn(k1, ..., -1)`：指定控制位输出低电平，对应继电器吸合
- 未在参数中指定的控制位输出高电平，对应继电器断开
- 参数列表必须以 `-1` 结束

常见理解：
- `SetOn(K1, K2, -1)`：K1、K2 吸合，其余断开
- `SetOn(-1)`：全部断开
- CBIT128 适合把“继电器状态表”集中表达，而不是逐个手动切

多工位使用注意：
- CBIT128 本身不做工位绑定
- 常见做法是按工位人为规划一组控制位：
  - SITE1: CBit0 / CBit1
  - SITE2: CBit8 / CBit9
  - SITE3: CBit16 / CBit17
  - SITE4: CBit23 / CBit24
- 然后一次 `SetOn(...)` 同时吸合多个工位的等价继电器

工程建议：
- 把继电器映射整理成宏或常量，避免裸写位号
- 多工位场景尽量按“每工位一组”规划，后续读代码和定位故障更轻松
- CON、Leakage、功能测试前的治具切换最好都通过固定继电器模板控制

常见风险：
- 忘记以 `-1` 结束参数列表，行为不可预期
- 误以为未指定的继电器会保持原状态，实际上 `SetOn` 会让未指定通道断开
- 多工位位号规划混乱，导致 site 间继电器互串
