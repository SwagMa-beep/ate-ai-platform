# STS8200S debug and waveform workflow

适用场景：
- 新工程首次联调
- 从 IDE/工程直接跳测试界面
- 软件示波器观察 `MeasureVI` 波形
- 保存测试数据和波形结果

从开发环境跳转到测试界面：
- 首次调试如果提示“系统找不到指定文件”，通常不是程序逻辑错，而是调试配置没配好
- 在工程属性里设置：
  - `Command`：指向 STS8200S 的 `testui`
  - `CommandArguments`：填写待测工程 `.pgs` 路径，并加 `-AutoLoad`

这样做的意义：
- 能直接从工程调试跳转到测试界面
- 避免每次手工再去找 PGS 装载

测试验证和数据保存流程：
1. 打开工作站 A
2. 点击“装载”导入 PGS
3. 在设置界面配置数据保存名称和路径
4. 执行单测
5. 测试完成后关闭或卸载，数据会自动按路径保存

软件示波器使用方法：
1. 装载 PGS 并先跑出一次结果
2. 打开“调试”
3. 打开“软件示波器”
4. 选择要观察的通道
5. 执行完 `MeasureVI` 之后，在示波器中观察对应通道波形
6. 可查看断点处的电压或电流值
7. 可保存：
   - 波形数据文件 `.svd`
   - 波形图片 `.bmp`

关键理解：
- 软件示波器不是无条件就有波形
- 只有执行了 `MeasureVI` 的通道，才会有可观察数据
- 所以“示波器没波形”首先要排查是不是测试流里根本没走到 `MeasureVI`

工程建议：
- 初次联调时，把“测试结果 + 示波器波形 + fail map”结合起来看
- 模拟参数异常时，用示波器确认实际激励和采样时序
- 动态测试建议保存 `.svd`，方便复盘
- 需要汇报或记录时再保存 `.bmp`

常见风险：
- 以为打开示波器就会自动有波形，忽略了 `MeasureVI`
- 调试配置里没有 `-AutoLoad`，导致每次跳过去都还是空工程
- 数据保存路径没配，跑完结果难复盘
