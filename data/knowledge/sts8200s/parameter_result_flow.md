# STS8200S 参数读取与结果上报闭环

目标：
说明 ATE 程序从“读取参数”到“上报结果”的最小闭环。

推荐流程：
1. 读取参数对象，例如 `StsGetParam(...)`
2. 从参数对象中获取上下限、名称、条件或结果句柄
3. 配置资源与量程
4. 执行测量
5. 调用 `SetTestResult(...)` 上报
6. 执行恢复动作，例如下电、清资源、复位通道

必须检查的点：
- 每个测试项是否都能找到对应参数对象
- 每个参数是否都进行了结果上报
- 结果单位是否与测试计划一致
- 多站点时是否按站点分别记录结果
- 失败分支是否也会产生明确结果，而不是直接 return

常见代码问题：
- 只读取参数但没有上报
- 只上报 PASS/FAIL，没有记录测量值
- 参数索引写死，后续计划变更后程序失效
- 保护触发时直接退出，没有写入 fail 结果
