# STS8200S module installation alignment

适用场景：
- 机台资源盘点
- 测试程序上机前确认模块安装与插位配置
- 判断“目标 site 数量是否真实可实现”

核心能力 / 限制：
- 插件箱可安装的模块数量和插位类型有限
- 模块是否能用，不仅取决于有没有板卡，还取决于它装在什么位置、是否与背板/总线匹配
- 多工位能力最终受模块数量、插位、稀缺资源和供电路径共同约束

安装与对位规则：
1. 建立机台实配清单
   - 记录模块名称、数量、插位、版本
2. 确认程序假设与机台一致
   - 程序里用到的 FOVI / DIO / MPOW / QTMU 数量不能只按标准配置假设
3. 确认高价值资源优先级
   - QTMU、FOVI、DIO、CBIT、多工位板通常是最容易成为瓶颈的资源
4. 确认 site 目标是否合理
   - 软件能开 4 site，不代表资源真的够支撑 4 site

与 TestPlan / ResourceMap 的关系：
- TestPlan 中每个关键测试项都应能映射到实际模块
- ResourceMap 中的通道编号、site 绑定和硬件安装位置应能互相对应
- 资源冲突有时不是逻辑冲突，而是实配数量不足

工程建议：
- 在 bring-up 前先做一次“模块安装与程序假设对照表”
- 对关键资源增加配置检查和日志记录
- 多工位方案先从保守工位数验证，再逐步放大
- 当 HardWareCfg 成功但实测异常时，优先回查安装位置和线缆拓扑

常见风险 / 注意事项：
- 程序按理想配置写，机台却是裁剪配置
- site 扩展过快，QTMU / FOVI / DIO 成为瓶颈
- 同一类模块数量足够，但安装位置或线缆分配不支持目标路径
