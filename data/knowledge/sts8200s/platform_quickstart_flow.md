# STS8200S platform quickstart flow

适用场景：
- 新工程上机前准备
- 竞赛或教学场景下的标准测试流程
- 从适配器接线到向量执行的最小闭环

典型上机流程：
1. 打开电脑和测试主机电源
2. 登录 STS8200S 软件
3. 等待系统初始化完成
4. 执行板卡自检，确认全部 PASS
5. 将机箱资源引出与适配器连接牢靠
6. 放置 DUT，并按芯片引脚定义设置 VCC / GND / 通道映射
7. 新建工程或装载现有 PGS
8. 准备向量文件、程序和测试条件
9. 执行测试验证并保存数据

平台基础资源认知：
- FOVI：基础模拟 VI 源，适合供电和参数测量
- DIO24：24 路数字通道，支持向量执行
- TMU：时间测量
- CBIT：继电器控制
- 通用数字适配器：支持 24PIN 器件，集成 VCC/GND 拨码与资源映射

工程实现最小闭环：
1. `UserLoad()` 里初始化 DIO 并加载 `.vecdio`
2. `HardWareCfg()` 确认工位和资源配置
3. `InitBeforeTestFlow()` 做源和板卡安全初始化
4. 根据测试项执行：
   - DIO `Run()`
   - FOVI / PMU `SetAndMeas()`
   - TMU 时间测量
5. `SetTestResult()` 写回参数
6. `InitAfterTestFlow()` 收尾

适配器接线要点：
- 插接必须牢靠，接触不良会直接影响连接性测试
- 数字通道与插座引脚有固定对应关系，必须按 datasheet pin 定义拨对
- 多电源器件要先确认 VCC1 / VCC2 使用策略

工程建议：
- 第一次跑新工程，先做自检 + CON，再做 FUN，再做参数测试
- 遇到异常时，先确认：
  - 板卡自检
  - 向量文件是否加载成功
  - 适配器 VCC/GND 拨码是否正确
  - DIO / FOVI / CBIT 路径是否对应

常见风险：
- 未自检就直接测试，导致把硬件问题误判成程序问题
- 向量文件没有和 PGS 放在同一路径
- 适配器拨码与芯片 pin 定义不一致
- 自以为程序错了，实际是适配器连接松动或接反
