# STS8200S 测试代码审查规则

审查重点：

1. 参数读取
- 每个测试项是否调用 `StsGetParam(...)`
- 参数名、索引、站点索引是否与 TestPlan 一致

2. 结果上报
- 每个测试项是否调用 `SetTestResult(...)`
- 失败分支是否也会产生有效结果

3. 初始化与恢复
- `HardWareCfg()` 是否负责全局资源配置
- `InitBeforeTestFlow()` 是否在每次测试前调用
- 结束后是否执行下电、释放和恢复动作

4. 量程与保护
- FOVI、QTMU、DMM 量程是否匹配目标参数
- 是否设置限流、保护和异常恢复

5. 资源映射
- 通道编号是否与 ResourceMap 一致
- 是否存在复用冲突或无效通道

典型问题：
- 参数读了但没上报
- 结果上报了但单位错了
- 通道写死，计划一变程序就错
- 没有恢复动作，导致后续测试串扰
