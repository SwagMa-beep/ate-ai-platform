# 良率波动排查手册

当良率波动时，优先按下面顺序排查：

1. 先看分布
- 是单一测试项波动，还是多项同时波动
- 是单一站点、单一机台，还是全线波动
- 是常温问题，还是高低温问题

2. 再看硬件
- 接触是否稳定
- 继电器是否老化
- 资源卡是否漂移
- 治具压降或线阻是否异常

3. 再看程序
- 最近是否改过量程、限值、延时、判定逻辑
- 是否存在初始化不完整
- 是否存在偶发不上报或异常退出

4. 再看样本
- 样本量是否足够
- 是否只有少量边界样本导致假波动
- 是否需要按 lot、site、温区重新分桶

常见经验：
- 如果只在高温失效，优先看温漂、负载和保护边界
- 如果只在某站点失效，优先看接触、继电器和 site 资源差异
- 如果多个项目一起变差，优先看供电、共地和初始化链路
