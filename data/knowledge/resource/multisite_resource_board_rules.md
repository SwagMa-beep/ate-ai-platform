# Multisite resource board rules

适用场景：
- 多工位模拟/混合信号测试
- LDO、EEPROM 等不同器件共用一块资源板
- 需要在不同器件或不同工位之间切换供电与测量路径

资源板的核心作用：
- 把机台内部资源引到板上，方便快速搭建测试电路
- 用继电器矩阵做资源切换和器件隔离
- 支持多工位并行测试，减少重复布线与程序切换成本

SH8280 资源板案例中的关键设计思路：
- 资源引出区：通过 64pin 线缆把内部资源带到板上
- 继电器控制区：通过 16 个干簧继电器切换不同 DUT 的供电路径
- 4 工位稳压器模块：适合 LDO 的 VO / 线性调整率 / 负载调整率 / Iq
- 4 工位 EEPROM 模块：适合 I2C 读写类测试

为什么要做继电器隔离：
- LDO 的输入电压和工作电流通常明显高于 EEPROM
- 若不切断供电通路，高压/大电流路径可能直接烧毁低压器件
- 所以多工位资源板不仅是“分配资源”，更重要的是“保护不同器件和工位”

资源规划原则：
1. 先按 DUT 类型分测试通路
   - 高压/大电流器件通路
   - 低压/弱电流器件通路
2. 再按工位做并行复制
3. 最后再决定哪些通路可以共享继电器控制模板

与 TestPlan / ResourceMap 的关系：
- TestPlan 如果没有明确区分不同器件的供电边界，资源板设计会很被动
- ResourceMap 不能只看“哪个 pin 接哪个资源”，还要看：
  - 这条路径是否经过继电器切换
  - 是否存在高压/高流和低压器件共线风险
  - 是否支持并行 site 的独立开关控制

工程建议：
- 在 ResourceMap 里增加“切换路径/继电器组”这一层，而不只是 pin-to-resource
- 对高压/大电流路径单独做保护标记
- 对每一类 DUT 先定义安全上电/断电顺序
- 多工位并行设计时，优先确保 site 间不会因为共享通路而互相干扰

常见风险：
- 只按通道数量分配资源，不考虑器件类型差异
- 用同一条供电路径服务高压 LDO 和低压 EEPROM
- 多工位名义独立，实际仍共用关键开关点
- ResourceMap 里没有表达继电器切换关系，导致程序实现与硬件不一致
