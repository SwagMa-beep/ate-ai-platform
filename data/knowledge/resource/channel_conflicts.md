# ResourceMap 通道冲突与实现性检查

目标：
帮助判断 ResourceMap 是否能真实落到治具和 STS8200S 资源上。

重点检查：
1. 同一测试步骤是否有多个动作争用同一资源
2. 高压与小信号测量是否走了不合适的共线路径
3. 大电流引脚是否与敏感模拟节点共用不合理资源
4. 温度传感器、DMM、SMU 等稀缺资源是否被重复占用

常见冲突：
- 同一引脚同时要求数字向量和模拟精测
- 高压输出和 DMM 精测共用路径，导致保护和精度都不理想
- 双工位方案下，没有预留隔离，导致 site 间串扰

建议动作：
1. 先按测试阶段拆开资源需求，不要把所有资源视为同时占用
2. 把高风险引脚单独列出来做隔离检查
3. 对稀缺资源建立优先级，避免在同一步骤里抢占
4. 对最终 ResourceMap 做一次逐项交叉验证，确认和 TestPlan 条件一致
