# Adapter relay topology patterns

适用场景：
- 数字/模拟芯片适配器设计
- 24PIN 通用适配器
- 多工位适配器
- 需要结合 DIO / FOVI / CBIT / TMU 的器件接口板

24PIN 数字适配器的典型资源结构：
- D0-D23：数字通道
- CBIT0-CBIT39：继电器控制位
- DGND / AGND / JGND：不同地参考
- J5V：固定 5V
- VDD / VSS：固定 ±15V
- FHx / SHx：FOVI 的 FORCE / SENSE
- TMU：时间测量辅助资源

典型供电路径：
- VCC1 通常由 FOVI0 提供
- VCC2 通常由 FOVI1 提供
- 外部上拉/下拉模块可由 FOVI2 提供偏置
- 通过拨码开关或继电器把这些源路由到 DUT 电源脚

数字适配器的常见拓扑特征：
- DIO 通道通过板上继电器与 DUT 数字脚接通/断开
- 继电器位号通常一一映射到管脚或管脚组
- 外部上拉/下拉模块单独使用一批继电器控制
- TMU 辅助模块通常通过额外路径接入需要时间测量的被测脚

模拟/混合信号适配器的常见拓扑特征：
- FOVI / TMU / CBIT 组合更常见
- 模拟源路径往往比数字路径更强调：
  - Kelvin
  - clamp
  - relay isolation
- 某些芯片（如 LDO）会同时存在：
  - 高压输入路径
  - 低压输出采样路径
  - 使能/时序触发路径

继电器拓扑设计经验：
- 把“测试资源切换”和“器件保护”一起考虑
- 能用继电器实现物理隔离的，不要只靠软件状态假设
- 对多电源、多模式器件，最好把供电、测量、时序路径分层控制

工程建议：
- 给每类继电器建立清晰的命名和映射表：
  - pin -> CBIT -> relay -> path -> resource
- 在程序和 ResourceMap 中同步保留这层关系
- 对同一个 pin 可能接不同资源的场景，先定义切换优先级和默认安全态

常见风险：
- 只知道 pin 对应资源，不知道中间经过哪些 relay
- 数字/模拟资源共线时没有隔离，导致误驱动或倒灌
- 上拉/下拉辅助通路忘记断开，影响后续精密测量
- 多工位适配器中某些 relay 实际共用，程序却按独立 site 处理
