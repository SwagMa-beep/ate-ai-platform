# Analog test principles

适用范围：
- 运算放大器、基准源、LDO 等模拟器件
- 直流精度、偏置、失调、增益、噪声、调整率等参数测试
- TestPlan 与测试条件设计审查

通用测试原则：
- 温度条件、参考点温度偏差必须满足详细规范
- 供电电压和其他激励量的精度必须在规范允许范围内
- 测试期间要抑制外界干扰对微小信号测量的影响
- 连接/断开被测器件时不得超过器件极限条件
- 有存储、滞后或预充电特性的模拟器件，需要明确预置条件
- 注意静电保护，尤其是高阻输入和精密参考类器件

模拟 TestPlan 的核心关注点：
1. 测试条件必须闭环
   - 温度
   - 输入电压 / 电流
   - 负载
   - 输出观察点
2. 限值来源必须清楚
   - datasheet 最小/典型/最大值
   - 是否考虑机台误差与 guard band
3. 量程选择必须匹配参数级别
   - 精密小电流、小电压不能用粗量程凑合
4. 预稳定时间必须明确
   - 上电后多久开始测
   - 动态波形后多久采样

典型模拟参数的工程含义：
- 输入失调 / 输出精度：对量程、零点漂移、采样噪声敏感
- 偏置 / 静态电流：对 guard、漏电、量程选择敏感
- 增益 / 调整率：对输入源稳定性、负载条件、采样方法敏感
- 噪声：对带宽、滤波、接地和屏蔽敏感

与平台实现的关系：
- 量程与 clamp 是模拟测试最先要审查的实现边界
- Kelvin、guard、继电器切换顺序会直接影响低电流和高精度测试
- 多工位时要确认每个 site 的模拟源和测量路径是否真正独立

工程建议：
- 对每个关键模拟参数都记录：
  - 刺激源
  - 测量源
  - 量程
  - 稳定时间
  - 限值依据
- 对高精度参数优先预留 guard band 和硬件误差裕量
- 对动态参数分开描述静态偏置条件和动态扫描条件

常见风险：
- 只写目标参数，不写负载/温度/输入条件
- 典型值直接当限值使用，没有考虑平台误差
- 用错误量程做精密测量
- 测试流程里没有明确预稳定时间
