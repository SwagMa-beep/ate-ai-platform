# Level translator test rules

适用场景：
- 电平转换器、跨电压域接口芯片的 TestPlan 审查
- 判断平台资源电平是否与器件输入/输出域兼容
- 评审方向控制、三态、高阻和跨域时序相关风险

核心能力 / 限制：
- 电平转换器同时存在输入侧和输出侧两个电平域，不能混为一谈
- 输入阈值、输出高低电平、方向控制和高阻行为通常都需要分别验证
- 平台资源和适配器必须清楚区分“低压侧”和“高压侧”的接入路径

典型测试项目思路：
- 电源与使能：确认双电源或多电源条件、使能逻辑与上电顺序
- 输入阈值：分别验证低压域/高压域输入条件
- 输出电平：分别验证转换后的 VOH / VOL 或边沿行为
- 方向控制：双向或方向可配器件需验证方向脚或自动方向策略
- 高阻与隔离：禁能状态、三态状态、总线共享场景下的泄漏和冲突

TestPlan 审查规则：
1. 电压域要分开写
   - 不要只写一个 VCC，而要区分各域供电
2. 输入 / 输出侧条件要分开写
   - VIH / VIL / VOH / VOL 应按不同电平域拆开
3. 方向控制要明确
   - 哪个方向、什么控制条件、测试时如何切换
4. 负载与保护要合理
   - 跨域输出不能用超出目标域的电压或不合理负载硬拉
5. 总线共享场景要说明
   - 若多个器件共享总线，需要明确高阻、上拉、下拉和冲突避免策略

工程建议：
- ResourceMap 中显式标注低压域、高压域和方向控制脚
- 程序实现上把不同方向的测试拆成独立步骤，避免条件混淆
- 对双向总线类器件，优先先验证高阻与方向切换，再做功能
- 在适配器评审时检查是否存在跨电压域误接风险

常见风险 / 注意事项：
- 把不同电平域的输入输出条件混写
- 方向脚状态没定义清楚
- 高阻状态没有验证，导致总线冲突
- 适配器或 ResourceMap 没有清楚区分两侧资源路径
