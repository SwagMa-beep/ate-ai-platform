# CMOS test condition rules

适用场景：
- CMOS 器件 TestPlan 制定与审查
- 静态电流、输入漏电、逻辑功能、电平阈值测试条件检查
- 评审低功耗或高阻输入器件的资源和治具策略

核心能力 / 限制：
- CMOS 输入阻抗高，对悬空、串扰、泄漏和外部偏置更敏感
- 逻辑功能看似简单，但输入端状态、负载、电源边界会明显影响结果
- 静态电流和漏电测试对量程、稳定时间和环境噪声要求更高

典型测试项目思路：
- 连接性：确认基础连线无误，但不宜用会过度应力的条件
- 功能：在明确 VDD、输入阈值和输出负载条件下检查逻辑行为
- 输入参数：VIH、VIL、输入漏电流、输入电容相关表现
- 输出参数：VOH、VOL、驱动能力、三态/高阻行为
- 电源相关：静态电流、待机功耗、动态功耗

TestPlan 审查规则：
1. 电源条件要完整
   - VDD 数值、容差、上电与掉电方式要明确
2. 输入端状态必须全部定义
   - 未参与测试的输入端不能悬空
3. 负载与量程要匹配
   - 高阻输出或小电流项目不能套用过粗量程
4. 稳定时间要足够
   - 漏电、静态电流、三态测量前要预留稳定时间
5. 温度与环境要说明
   - 对低漏电、低功耗器件，温度和治具清洁度会明显影响结果

工程建议：
- 如果是 CMOS 门电路或低功耗器件，优先把“输入端固定规则”写进模板
- 对静态电流和漏电项目，优先检查资源量程、接触质量和治具泄漏
- 对三态、高阻类器件，增加输出隔离和外部偏置说明
- 把电源、输入、负载、稳定时间写成完整测试条件，不要只留参数名

常见风险 / 注意事项：
- 输入端悬空导致功能或电流结果漂移
- 漏电/静态电流使用过大电流量程
- 稳定时间不足导致结果不重复
- 治具污染、潮湿或继电器泄漏被误判为器件异常
