# Digital test principles

适用范围：
- TTL、CMOS、Level Translator 等数字器件
- 连接性测试、功能测试、静态电参数测试
- TestPlan 审查与数字模板参数设置

测试期间的通用原则：
- 环境温度或参考点温度必须符合器件详细规范
- 应避免外界干扰影响测试精度
- 电源电压应保持在规定值附近，其他电参量准确度应满足规范
- 被测器件接入或断开测试系统时，不得超过器件极限条件
- 有内部存储或滞后效应的器件，应规定预置条件
- 非被测输入端不得悬空
- 测试期间要避免 ESD 损伤

数字器件 TestPlan 审查重点：
1. 电平条件完整
   - VIH / VIL / VOH / VOL 必须明确
   - 不能只写“高电平”“低电平”而没有实际数值
2. 电源条件完整
   - VCC 值、量程、负载边界要明确
3. 向量条件完整
   - 起始 label / 结束 label / 时间集要能对应向量文件
4. 非被测端状态明确
   - 输入不能悬空
   - 开漏/三态器件要说明外部上拉/下拉

TTL / CMOS 共通的工程逻辑：
- 功能测试前通常先做连接性测试
- 功能测试若不通过，后续很多参数测试失去意义
- 测试电平不应只“能跑”，而要与 datasheet 逻辑门限一致
- 对供电边界、输出负载、输入偏置的假设必须明写

Level Translator 额外关注点：
- 输入侧与输出侧电平域必须区分
- 输出高低电平及其阈值需要分别验证
- 高阻态电流与方向性行为要特别关注

连接性测试原则：
- 用于确认引脚与测试系统之间已建立电气连接
- 用于尽早剔除：
  - 引脚短路
  - bond wire 缺失
  - ESD 损伤
  - 制造缺陷
- 是降低平均测试时间的重要前置步骤

工程建议：
- 数字器件模板里先把 GlobalVariable、pin 分组、vecdio、VCC 条件配准
- 再逐步补 CON、FUN、FIMV_PMU、FVMI_PMU
- 对开漏、三态、双电源器件单独标记外围条件

常见风险：
- 输入脚悬空
- VCC 和逻辑电平没有按 datasheet 成对设置
- 向量 label 与 PGS 条件不一致
- 把所有参数都挤进一个功能模式里，无法区分问题来自连接、电平还是向量本身
