# TTL test condition rules

适用场景：
- TTL 器件 TestPlan 制定与审查
- 数字器件 CON / FUN / 静态参数测试条件检查
- 像 HD74LS00P 这类 TTL 门电路的程序与条件评审

核心能力 / 限制：
- TTL 测试要同时关注逻辑功能、电平阈值、输入电流、输出驱动和电源电流
- 测试条件不能只写“高/低电平”，应给出明确电压和电流边界
- 非被测输入端不能悬空，否则功能和静态参数都会失真

典型测试项目思路：
- 连接性：先确认引脚、电源、地和关键 IO 已建立电气连接
- 功能：按真值表或向量文件检查逻辑行为
- 输入参数：VIH、VIL、IIH、IIL、VIK 等
- 输出参数：VOH、VOL、IOS 等
- 电源相关：ICC、待机/工作电流
- 时序相关：若器件有传播延时要求，再补充 TPLH / TPHL 等

TestPlan 审查规则：
1. 电源条件必须明确
   - VCC 数值、允许偏差、上电顺序要明确
2. 输入条件必须明确
   - VIH / VIL 不能只写逻辑 1 / 0
   - 未参与当前测试的输入端要定义固定状态
3. 输出负载必须明确
   - VOH / VOL 测试需要写清负载电流方向与大小
4. 功能向量必须可追溯
   - 向量 label、模式、速度与预期功能要对应
5. 参数来源必须可追溯
   - 限值要来自 datasheet 或明确标准

工程建议：
- 数字器件程序通常先做 CON，再做 FUN，再做直流参数，最后做时序
- 如果 FUN 不通过，优先排查引脚连接、电源、输入固定状态，再看参数测试
- 对 TTL 器件，输入上拉/下拉和外部负载设定要尤其谨慎
- 对批量生产方案，建议把“输入不悬空”和“输出负载合法”作为固定审查项

常见风险 / 注意事项：
- 非被测输入脚悬空
- VCC 与逻辑电平条件不匹配
- VOH / VOL 负载方向写反
- 把功能不通过误判成器件参数不合格
- 时序测试没先确认功能与连接性
