# Op-amp test method overview

适用场景：
- 运算放大器 TestPlan 制定与审查
- 模拟器件测试项优先级排序
- 判断 STS8200S 上实现运放测试时该优先考虑哪些条件

覆盖对象：
- 电压型运算放大器
- 电流型运算放大器
- 单路与多路运放 DUT

标准中高价值的测试主题：
- 输入失调电压
- 输入偏置电流 / 输入失调电流
- 静态功耗
- 开环增益 / 传输阻抗
- 共模抑制比
- 电源电压抑制比
- 转换速率
- 建立时间
- 输出峰峰电压
- 最大输出电流 / 输出短路电流
- 带宽、增益带宽积、谐波失真
- 噪声、通道隔离度、输入输出阻抗

通用测试条件原则：
1. 环境条件要符合器件详细规范
   - 温度、参考点温度、外界干扰控制要明确
2. 电源与激励要准确
   - Vs+ / Vs- 以及其他激励条件精度要满足规范
3. 连接与断开不得越过极限条件
   - 上电顺序、断电顺序、输入输出应力要受控
4. 辅助电路要明确
   - 补偿网络、辅助电路、负载和测试环路要写清
5. 防自激与静电保护要考虑
   - 运放类器件特别容易被环路、补偿和噪声影响

辅助运放（AUX）思路：
- 标准里明确出现“带 AUX 的闭环测试方法”这一类思路
- 对高速或宽带运放，AUX 有助于降低自激和测量不稳定风险
- AUX 自身必须有足够的开环增益、合适的输入偏置和独立供电，不应反过来污染 DUT 结果

TestPlan 审查建议：
1. 先确认项目目标是直流精度、动态响应还是频域特性
2. 把供电、负载、补偿网络和稳定时间写成完整测试条件
3. 对失调、偏置、电流类项目优先审查量程与泄漏路径
4. 对 SR / 建立时间 / 带宽类项目优先审查激励方式、采样链路和自激风险
5. 对噪声、失真类项目单独审查外界干扰和测量带宽定义

工程建议：
- 在 STS8200S 上先把运放测试分成直流、动态、频域三大类，再映射资源
- 小信号/微弱电流项目优先确认量程、稳定时间、夹具泄漏和屏蔽
- 对高速项目，优先确认是否需要 AUX、补偿网络和更严格的布线约束
- 把“是否可能自激”列为运放方案评审的固定问题

常见风险 / 注意事项：
- 只定义参数名，不定义供电、负载和补偿条件
- 把宽带/高速项目当普通 DC 项处理
- 忽略 AUX 和闭环测试结构本身带来的误差
- 未控制外界干扰、泄漏和上电顺序，导致结果不稳定
