# 课题实施方案报告

## 1. 课题名称

基于语言大模型的智能 ATE 测试开发辅助平台。

## 2. 研究背景

在集成电路测试开发过程中，工程师通常需要从芯片 Datasheet 中人工提取电气参数、引脚定义、工作条件和测试限制，再结合 ATE 平台资源编写 TestPlan、资源映射表和测试程序。传统流程存在以下问题：

- Datasheet 内容复杂，表格、图表、跨页参数较多，人工提取耗时。
- 不同芯片类型的测试项和参数格式差异明显，复用难度高。
- 资源映射和测试代码编写依赖工程经验，初版错误率较高。
- 测试开发、调试和诊断数据分散，缺少统一的 AI 辅助平台。

因此，本课题拟构建一个面向 ATE 测试开发流程的 AI 辅助平台，利用大语言模型、规则校验、RAG 检索和轻量化诊断算法，提升测试开发效率。

## 3. 建设目标

本项目目标是实现从 Datasheet 到 ATE 测试开发交付物的智能辅助流程，主要包括：

1. 自动解析 Datasheet PDF，提取测试参数和引脚定义。
2. 根据 STS8200S 平台约束进行参数分类、范围校验和兼容性提示。
3. 生成 TestPlan Excel/JSON 文件，辅助测试工程师复核。
4. 根据引脚定义生成资源映射、PGS 配置、BOM 和辅助示意图。
5. 基于模板规则和 RAG 知识生成测试程序骨架。
6. 提供轻量化良率诊断演示模块。
7. 支持 Windows 桌面端安装运行，降低演示和部署门槛。

## 4. 系统总体方案

系统采用前后端分离架构：

- 前端：React + TypeScript + Electron，负责交互界面、任务状态展示、结果下载和桌面端封装。
- 后端：FastAPI + Python，负责 PDF 解析、LLM 调用、资源映射、代码生成和诊断分析。
- AI 能力：使用 DeepSeek API 进行结构化参数提取和代码生成；使用内置 STS8200S 知识片段进行 RAG 增强。
- 文件输出：生成 Excel、JSON、SVG 等工程交付文件。
- 桌面部署：使用 PyInstaller 打包后端运行时，Electron 自动启动内置后端。

## 5. 技术路线

1. PDF 解析：使用 pdfplumber / PyMuPDF 读取 Datasheet 文本和表格内容。
2. 页面过滤：通过关键词、文本密度和页面特征跳过封面、目录、修订记录等低价值页面。
3. 大模型提取：将有效文本块提交给 DeepSeek，约束输出结构化 JSON。
4. 结果清洗：对模型输出进行字段标准化、数值转换、单位归一和重复项合并。
5. 平台校验：根据 STS8200S 电压、电流、DIO 和 CBIT 等限制生成兼容性建议。
6. 资源映射：根据芯片类型、引脚方向和测试项映射平台资源。
7. 代码生成：通过模板 + LLM 注释补全生成测试代码骨架，并进行静态规则检查。
8. 良率诊断：基于仿真 VI 波形构建异常检测和故障分类演示。

## 6. 阶段计划

| 阶段 | 主要任务 | 预期产出 |
|---|---|---|
| 第一阶段 | 需求分析、架构设计、技术选型 | 课题实施方案、系统架构图 |
| 第二阶段 | PDF 解析、LLM 提取、TestPlan 生成 | 参数提取 API、Excel/JSON 输出 |
| 第三阶段 | 资源映射、代码生成、诊断模块 | 资源映射表、代码生成页面、诊断页面 |
| 第四阶段 | 性能优化、桌面端打包、测试验证 | Windows 安装包、测试数据报告 |
| 第五阶段 | 文档整理、视频录制、答辩准备 | 结题报告、演示视频、答辩 PPT |

## 7. 当前实现基础

项目目前已完成核心功能闭环，主要包括：

- Datasheet 上传和异步提取任务。
- PDF 页面过滤和本地缓存。
- DeepSeek 结构化参数提取。
- 引脚定义提取和部分图像 pinout 兜底。
- TestPlan Excel/JSON 导出。
- STS8200S 兼容性校验和量程建议。
- 资源映射、BOM、SVG 辅助示意图生成。
- C++ 测试代码生成和静态校验。
- 良率诊断演示。
- Windows 桌面安装包。

## 8. 后续优化方向

- 建立标准测试集，量化 Datasheet 识别准确率。
- 提升图像型表格、pinout 和扫描版 PDF 的识别能力。
- 完善 DUT 原理图辅助设计建议。
- 增强测试代码编译检查和真实平台适配。
- 将良率诊断接入真实机台数据。
- 建立云端后端，集中管理 API Key、日志、用户和任务。
