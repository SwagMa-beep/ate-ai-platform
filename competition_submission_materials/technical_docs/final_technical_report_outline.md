## 摘要

介绍项目背景、目标、核心方法、实现成果和指标完成情况。

## 1. 项目背景与意义

- ATE 测试开发流程现状。
- Datasheet 人工解析和测试程序开发痛点。
- 大语言模型在测试开发辅助中的应用价值。

## 2. 赛题指标与项目目标

列出赛题具体指标：

1. 大模型对 Datasheet 中复杂表格和图表的识别准确率 ≥85%。
2. DUT 原理图辅助设计建议准确率 ≥75%。
3. 测试程序辅助编程，自动生成代码符合测试要求，具备可实现性。
4. 工控机上运行轻量化 ATE 型，在不影响机台运行前提下完成实时诊断。
5. 相比传统开发模式，测试程序开发周期缩短不低于 50%，代码首检错误率降低 70% 以上。

- LLM 与 RAG 能力层。

## 8. 创新点

1. 将 Datasheet 解析、ATE 资源映射和测试代码生成整合为一体化流程。
2. 使用 LLM + 本地规则兜底提升工程可用性。
3. 面向 STS8200S 平台进行规则约束和资源映射。
4. 支持桌面端安装运行，降低演示和使用门槛。
5. 将测试开发辅助和良率诊断展示结合，覆盖开发与量产场景。

## 9. 不足与改进方向

- 客户级使用需云端部署。
